**CachePuppy** combines **realtime messaging** and a **distributed cache** in one system you run yourself (or with your vendor). The service is built on the **BEAM** (Erlang VM) using **Elixir**, so scaling is a matter of adding nodes: processes migrate, sockets stay coherent across the cluster, and cache partitions follow the same routing rules wherever your code touches them.

Use this site to:

- **Ship quickly** — [Quick start](/docs/quick-start) with Docker and a short JavaScript example.
- **Understand behavior** — [Core concepts](/docs/core-concepts) for channels, envelopes, topic state, session state, and cache semantics.
- **Integrate from JS** — [JavaScript](/docs/javascript) covers **`@cachepuppy/core`**, **`@cachepuppy/react`**, and the **admin HTTP** client.
- **Look up contracts** — [API reference](/docs/api-reference) for HTTP paths, websocket events, and exports.
