## Tooling

- **Elixir** compatible with `~> 1.15` and **Mix** (see `cachepuppy_core/mix.exs`).
- **Node.js** with **npm** for the documentation site and JavaScript SDK packages.
- A modern browser if you are testing websocket flows manually.

## Ports and URLs

With `mix phx.server`, the Phoenix app listens on **`http://localhost:4000`** by default.

The JavaScript SDK examples in this documentation assume a websocket URL like:

```text
ws://localhost:4000/socket/websocket
```

## Cluster environment variables

The Elixir application expects **`TOTAL_NODES`** to be set to a positive integer for quorum-aware behavior. For a comfortable local experience, export:

```bash
export TOTAL_NODES=1
```

You can tune optional `QUORUM_POLL_INTERVAL_MS` and `QUORUM_GRACE_MS` values later; see [Operations](/docs/core-concepts/operations) for what they mean.
